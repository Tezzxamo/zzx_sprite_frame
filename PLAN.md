# 视频转序列帧工具 — 产品开发计划

## 一、产品定位与目标用户

**产品名称**：SpriteFrame（精灵帧提取器）

**核心定位**：面向游戏开发者（尤其是独立开发者）的视频转序列帧桌面工具，专注于角色帧动画工作流优化，原生支持 Godot、Unity 等引擎的精灵动画制作管线。

**目标用户**：
- 使用 AI 生成视频/实拍素材制作 2D 角色动画的独立游戏开发者
- 需要将视频参考转为像素画/手绘动画参考的美术人员
- 使用 Godot AnimatedSprite 或 SpriteFrames 资源的开发者

## 二、功能需求完善（基于原始需求 + Godot 工作流增强）

### 2.1 原始需求功能清单

| 模块 | 功能点 | 优先级 |
|------|--------|--------|
| 视频导入 | 支持点击上传 / 拖拽上传 | P0 |
| 视频信息 | 显示分辨率、时长、原始帧率、编码格式、文件大小 | P1 |
| 画面裁剪 | 可视化画布裁剪（矩形选区） | P0 |
| 画面裁剪 | 左右双栏对比：左侧原画 + 右侧裁剪预览 | P0 |
| 画面裁剪 | 鼠标拖拽调整裁剪框 + 右侧参数面板微调（X/Y/Width/Height） | P0 |
| 帧提取 | 自定义每秒提取帧数（FPS） | P0 |
| 帧提取 | 实时计算并显示总帧数、预估输出大小 | P1 |
| 帧提取 | 支持输出格式：PNG、JPEG、WebP、BMP | P1 |

### 2.2 针对 Godot 角色动画的增强功能

| 模块 | 功能点 | 优先级 | 说明 |
|------|--------|--------|------|
| **精灵表导出** | 将序列帧合并为 Sprite Sheet（精灵表） | P1 | Godot AnimatedSprite 支持 SpriteSheet 导入，大幅减少资源文件数量 |
| **精灵表导出** | 支持自定义行列数、间距（Padding）、边框 | P2 | 防止纹理 bleeding |
| **透明背景** | 绿幕/蓝幕抠图 + 自定义颜色抠图 | P2 | 角色动画常见需求，去除视频背景 |
| **帧管理** | 提取后帧列表展示，支持删除/恢复/重新排序 | P1 | 去除废帧、调整动作顺序 |
| **智能去重** | 基于相似度算法自动标记/删除重复帧 | P2 | 减少冗余，优化动画流畅度 |
| **输出缩放** | 裁剪后支持按百分比或绝对像素缩放输出 | P1 | 方便控制游戏中精灵尺寸 |
| **命名模板** | 自定义帧文件命名规则（如 `run_{0001}.png`） | P1 | 支持 Godot 导入命名习惯 |
| **Godot 预设** | 一键导出 Godot SpriteFrames 资源文件（`.tres`） | P2 | 直接生成 Godot 可识别资源 |
| **项目保存** | 保存当前裁剪/提取配置为项目文件，支持重新打开调整 | P2 | 非破坏性工作流，方便迭代 |
| **参考线** | 画布支持九宫格/中心点/自定义参考线 | P2 | 帮助对齐角色锚点 |
| **批量处理** | 支持批量导入多个视频，统一配置提取 | P3 | 大量角色动画场景 |
| **快捷键** | 空格播放/暂停、方向键逐帧、Ctrl+S 保存项目 | P2 | 提升操作效率 |

## 三、技术架构方案

### 方案对比

| 维度 | 方案 A：Electron + React + FFmpeg | 方案 B：Tauri + React + FFmpeg |
|------|-----------------------------------|-------------------------------|
| **技术栈** | Electron + React + TypeScript + fluent-ffmpeg | Tauri + React + TypeScript + FFmpeg CLI |
| **安装包体积** | ~150-200MB | ~15-30MB |
| **性能** | 中（Chromium 内存占用较高） | 高（Rust 后端，内存低） |
| **开发效率** | 高（生态成熟，文档丰富） | 中高（较新，部分边界情况处理复杂） |
| **视频处理** | 通过 Node.js 调用 FFmpeg | 通过 Rust Command 调用 FFmpeg |
| **安全性** | 一般 | 高（Tauri 默认安全策略严格） |
| **维护成本** | 低 | 中 |

### 推荐方案：方案 A（Electron + React + FFmpeg）

**理由**：
1. 开发速度快，可快速验证 MVP
2. FFmpeg Node.js 封装（fluent-ffmpeg）成熟稳定
3. 团队招聘/维护成本低
4. 独立开发者工具，对安装包体积敏感度低于企业级应用

**系统架构**：

```
┌─────────────────────────────────────────────────────────────┐
│                        渲染进程 (Renderer)                    │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────────────┐  │
│  │ 视频导入模块 │  │ 裁剪预览模块 │  │ 帧提取与导出模块     │  │
│  │ - 拖拽上传   │  │ - 双栏画布   │  │ - FPS/格式/缩放设置 │  │
│  │ - 视频信息   │  │ - 参数微调   │  │ - 帧列表管理        │  │
│  └─────────────┘  └─────────────┘  └─────────────────────┘  │
│                        React + TypeScript + Fabric.js/Canvas │
├─────────────────────────────────────────────────────────────┤
│                      主进程 (Main)                            │
│  ┌─────────────────┐  ┌─────────────────────────────────────┐│
│  │ 文件系统操作     │  │ FFmpeg 视频处理服务                  ││
│  │ - 读写帧文件    │  │ - 视频解码/裁剪/抽帧/精灵表合并      ││
│  │ - 项目文件存取  │  │ - 格式转换/缩放/透明通道处理         ││
│  └─────────────────┘  └─────────────────────────────────────┘│
│                        Electron + Node.js + fluent-ffmpeg    │
└─────────────────────────────────────────────────────────────┘
```

## 四、UI/UX 设计规划

### 4.1 整体布局

采用 **三栏式布局**：

```
┌──────────────────────────────────────────────────────────────────┐
│  顶部工具栏：导入视频 | 保存项目 | 导出设置 | 关于                 │
├──────────────┬──────────────────────────────┬────────────────────┤
│              │                              │                    │
│  左侧面板     │      中间预览区域             │    右侧面板        │
│  - 视频信息   │   ┌─────────┐  ┌─────────┐  │  - 裁剪参数        │
│  - 提取设置   │   │ 原始画面 │  │裁剪预览 │  │  - 输出设置        │
│  - 帧率控制   │   │ (带选框) │  │         │  │  - 帧列表          │
│  - 输出格式   │   └─────────┘  └─────────┘  │  - 导出按钮        │
│              │                              │                    │
└──────────────┴──────────────────────────────┴────────────────────┘
```

### 4.2 关键交互设计

1. **裁剪交互**：
   - 中间左侧画布显示视频当前帧，叠加可拖拽矩形裁剪框
   - 拖拽四角/四边调整大小，拖拽中间移动位置
   - 右侧参数面板实时同步 X/Y/W/H，支持键盘方向键微调（±1px）
   - 按住 Shift 保持比例拖拽

2. **预览交互**：
   - 左右画面同步：播放视频时，右侧实时显示裁剪后的效果
   - 支持单帧前进/后退（← → 键或按钮）
   - 裁剪参数变化时，右侧预览即时更新

3. **提取流程**：
   - 设置 FPS → 自动计算总帧数 → 点击"提取帧"
   - 提取完成后，右侧面板切换为"帧列表"视图
   - 支持框选删除、Ctrl+Z 撤销删除
   - 点击"导出"选择文件夹，批量保存

## 六、技术细节补充

### 6.1 关键技术选型

| 模块 | 技术选型 | 说明 |
|------|----------|------|
| 前端框架 | React 18 + TypeScript | 类型安全，生态丰富 |
| 状态管理 | Zustand | 轻量，适合工具类应用 |
| UI 组件库 | Ant Design 或 shadcn/ui | 成熟稳定，支持暗色模式 |
| 画布操作 | Fabric.js 或原生 Canvas API | Fabric.js 更适合交互式选区 |
| 视频预览 | HTML5 Video 元素 | 简单高效，配合 Canvas 叠加选区 |
| 视频处理 | FFmpeg（通过 fluent-ffmpeg 调用） | 功能全面，社区成熟 |
| 打包构建 | electron-builder | 支持 Windows/Mac/Linux 自动打包 |

### 6.2 视频处理核心命令示例

```bash
# 裁剪 + 按 FPS 抽帧
ffmpeg -i input.mp4 -vf "fps=12, crop=400:400:100:50, scale=200:200" -q:v 2 output_%04d.png

# 生成精灵表（ImageMagick 或 ffmpeg 拼接）
ffmpeg -i input.mp4 -vf "fps=12, crop=400:400:100:50, tile=8x8" spritesheet.png
```

### 6.3 数据流设计

```
视频文件路径
    ↓
[主进程] FFmpeg 提取视频基本信息（时长、分辨率、帧率）
    ↓
[IPC] 传递到渲染进程
    ↓
[渲染进程] 用户设置裁剪参数 + 提取配置
    ↓
[IPC] 传递到主进程
    ↓
[主进程] FFmpeg 批量抽帧到临时目录
    ↓
[IPC] 帧列表 + 缩略图路径传递到渲染进程
    ↓
[渲染进程] 用户确认/删除帧
    ↓
[IPC] 导出指令
    ↓
[主进程] 从临时目录复制/合并为最终输出
```

## 七、风险与应对

| 风险 | 影响 | 应对策略 |
|------|------|----------|
| 大视频（>1GB）处理内存溢出 | 高 | 采用流式处理，逐帧提取而非全量加载；提供进度条 |
| FFmpeg 跨平台兼容性 | 中 | 打包时针对不同平台内置对应 FFmpeg 二进制文件 |
| Canvas 大分辨率视频卡顿 | 中 | 预览时使用降采样版本，提取时用原分辨率 |
| 不同视频编码格式支持 | 低 | 依赖 FFmpeg，默认支持绝大多数格式 |
